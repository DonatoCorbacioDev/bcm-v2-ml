# Dataset Sources — BCM ML Risk Scoring

## Current approach: supervised synthetic data

Real labeled contract datasets are not publicly available at the granularity needed
(contract value + expiry + outcome). We generate a synthetic dataset calibrated to
realistic contract portfolio distributions, label each record using domain-derived
rules, and train a classifier that can generalize beyond those rules via feature
interactions.

The synthetic generator is deterministic (fixed seed) so results are reproducible.

### Synthetic dataset schema

File: `data/synthetic_contracts.csv` (generated by `scripts/generate_training_data.py`)

| Column | Type | Description |
|---|---|---|
| `days_until_expiry` | float | Days from today to end_date; negative = expired |
| `status_code` | int | 0=ACTIVE, 1=EXPIRED, 2=CANCELLED, 3=DRAFT |
| `has_end_date` | int | 1 if end_date is set, 0 otherwise |
| `total_financial_amount` | float | Sum of all financial_values for the contract (EUR) |
| `num_financial_records` | int | Number of monthly financial records |
| `financial_std` | float | Standard deviation of monthly amounts |
| `financial_zscore` | float | Z-score of total amount within the organisation |
| `risk_level` | int | Ground truth: 0=LOW, 1=MEDIUM, 2=HIGH |

### Label derivation rules (ground truth)

```
HIGH (2):  status == EXPIRED  OR  days_until_expiry < 30  OR  |z_score| > 2.5
MEDIUM (1): days_until_expiry < 90  OR  |z_score| > 1.5
LOW (0):   otherwise
```

5% label noise is added to simulate labelling uncertainty and prevent the model
from perfectly memorising the rule boundary.

### Distribution targets

| Class | Target share |
|---|---|
| LOW | ~40% |
| MEDIUM | ~35% |
| HIGH | ~25% |

## Future real data sources

### Italian public procurement (ANAC)

- **Source:** [dati.anticorruzione.it](https://dati.anticorruzione.it/)
- **Data:** CIG (Codice Identificativo Gara) open data — procurement lots, amounts,
  contracting authorities, awarded companies, execution dates.
- **Relevance:** real contract amounts and timeline data; no explicit "risk" label
  but delayed execution, rebids, and cancellations can proxy for contract stress.
- **Limitation:** public sector only; private B2B contracts not covered.

### Banca d'Italia — Central Credit Register

- **Source:** [bancaditalia.it/statistiche](https://www.bancaditalia.it/statistiche/)
- **Data:** firm-level credit exposure and default rates by sector and size class.
- **Relevance:** financial stress signals at counterparty level — a customer with
  high credit risk is more likely to default on a contract.
- **Limitation:** aggregate statistics, not contract-level; requires linking customer
  name to firm registry (Registro Imprese via API Camere di Commercio).

### EU Funding (Open Data Portal)

- **Source:** [data.europa.eu](https://data.europa.eu/)
- **Data:** Horizon Europe and Cohesion Fund contract awards with amounts and durations.
- **Relevance:** shows realistic distribution of public contract values, durations,
  and multi-year structure.

### Synthetic calibration path (current)

Until real labeled data is available the workflow is:

```
generate_training_data.py   →  data/synthetic_contracts.csv
train_risk_model.py         →  model/risk_model.joblib
                                model/risk_model_metadata.json
FastAPI startup             →  loads model, serves /risk-scores with ml_score field
```

When real data becomes available, replace `data/synthetic_contracts.csv` with real
records (same schema), re-run `train_risk_model.py`, and the rest of the pipeline
is unchanged.
